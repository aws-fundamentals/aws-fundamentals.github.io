What is Amazon EC2?

Amazon EC2 (Elastic Compute Cloud) is a web service that provides scalable compute capacity in the cloud. It allows you to run virtual servers (instances) on-demand, which you can configure based on your needs. Think of EC2 as renting a computer in the cloud that you can turn on and off as you need.

Key Concepts

Instances

  • Instance Types: Various types optimized for different use cases, such as compute-optimized and memory-optimized.
  • AMIs (Amazon Machine Images): Templates with pre-configured setups for your instance.
  • Instance States: Instances can be running, stopped, terminated, etc.

Storage

  • EBS (Elastic Block Store): Persistent storage that remains even if you stop or terminate your instance.
  • Instance Store: Temporary storage that is lost if the instance is stopped or terminated.
  • S3 (Simple Storage Service): Used for storing large amounts of data and backups.

Networking

  • Security Groups: Virtual firewalls controlling traffic to and from instances.
  • Key Pairs: Used for SSH or RDP access to instances.
  • Elastic IPs: Static IP addresses for dynamic cloud computing.
  • VPC (Virtual Private Cloud): A network you define within AWS to isolate and control your resources.

Getting Started with EC2

Step 1: Launching Your First Instance

  1. Sign in to AWS Management Console: Go to the AWS Management Console and log in.
  2. Open EC2 Dashboard: Navigate to Services and click on EC2.
  3. Launch Instance:
    • Choose an AMI: Select a pre-configured image like Amazon Linux 2 AMI or Ubuntu.
    • Choose an Instance Type: Select a type based on your needs (e.g., t2.micro).
    • Configure Instance: Modify settings as needed.
    • Add Storage: Use default storage or add more.
    • Add Tags: Add tags to organize your instances.
    • Configure Security Group: Define rules to control traffic.
    • Review and Launch: Review your settings and launch the instance. Save your private key file.
  4. Connect to Your Instance: Follow instructions to connect using SSH or an SSH client.

Step 2: Managing Your Instance

  • Stop and Start: Save costs by stopping instances when not in use.
  • Terminate: Permanently delete instances when no longer needed.

Step 3: Security and Networking

  • Security Groups: Define rules to control access.
  • Key Pairs: Securely connect to instances.
  • VPC: Create a custom VPC for more control over your network.

Additional Features

  • Elastic Load Balancing (ELB): Distributes incoming traffic across multiple instances.
  • Auto Scaling: Automatically adjusts the number of instances based on demand.
  • Spot Instances: Bid on unused EC2 capacity at lower prices.
  • Reserved Instances: Discounted pricing for committing to use EC2 for a term.
  • Savings Plans: Flexible pricing model with savings based on usage commitment.

আমাজন EC2 কী?

আমাজন EC2 (Elastic Compute Cloud) হল একটি ওয়েব সার্ভিস যা ক্লাউডে স্কেলেবল কম্পিউট ক্ষমতা প্রদান করে। এটি আপনাকে অন-ডিমান্ডে ভার্চুয়াল সার্ভার (ইনস্ট্যান্স) চালাতে দেয়, যা আপনি আপনার প্রয়োজন অনুযায়ী কনফিগার করতে পারেন। EC2-কে ক্লাউডে একটি কম্পিউটার ভাড়া নেওয়ার মতো ভাবুন, যা আপনি প্রয়োজন অনুযায়ী চালু এবং বন্ধ করতে পারেন।

মূল ধারণাসমূহ

ইনস্ট্যান্স

  • ইনস্ট্যান্স টাইপস: বিভিন্ন প্রয়োজনে অপ্টিমাইজড বিভিন্ন ধরনের ইনস্ট্যান্স, যেমন কম্পিউট-অপ্টিমাইজড এবং মেমরি-অপ্টিমাইজড।
  • AMIs (Amazon Machine Images): আপনার ইনস্ট্যান্সের জন্য প্রি-কনফিগারড সেটআপ সহ টেমপ্লেট।
  • ইনস্ট্যান্স স্টেটস: ইনস্ট্যান্স চালু, বন্ধ, মুছে ফেলা ইত্যাদি অবস্থায় থাকতে পারে।

স্টোরেজ

  • EBS (Elastic Block Store): স্থায়ী স্টোরেজ যা আপনার ইনস্ট্যান্স বন্ধ বা মুছে ফেলা হলেও থাকে।
  • ইনস্ট্যান্স স্টোর: অস্থায়ী স্টোরেজ যা ইনস্ট্যান্স বন্ধ বা মুছে ফেলা হলে হারিয়ে যায়।
  • S3 (Simple Storage Service): বড় পরিমাণে ডেটা এবং ব্যাকআপ সংরক্ষণের জন্য ব্যবহৃত হয়।

নেটওয়ার্কিং

  • সিকিউরিটি গ্রুপস: ইনস্ট্যান্সের দিকে এবং থেকে ট্রাফিক নিয়ন্ত্রণকারী ভার্চুয়াল ফায়ারওয়াল।
  • কী পেয়ার্স: ইনস্ট্যান্সে SSH বা RDP অ্যাক্সেসের জন্য ব্যবহৃত হয়।
  • ইলাস্টিক আইপিস: গতিশীল ক্লাউড কম্পিউটিংয়ের জন্য স্ট্যাটিক IP অ্যাড্রেস।
  • VPC (Virtual Private Cloud): AWS-এর মধ্যে একটি নেটওয়ার্ক যা আপনি আপনার রিসোর্সগুলি আলাদা এবং নিয়ন্ত্রণ করার জন্য সংজ্ঞায়িত করেন।

EC2 দিয়ে শুরু করা

ধাপ ১: আপনার প্রথম ইনস্ট্যান্স চালু করা

  1. AWS ম্যানেজমেন্ট কনসোলে সাইন ইন করুন: AWS Management Console এ যান এবং লগ ইন করুন।
  2. EC2 ড্যাশবোর্ড খুলুন: সার্ভিসেস-এ যান এবং EC2 তে ক্লিক করুন।
  3. ইনস্ট্যান্স চালু করুন:
    • AMI নির্বাচন করুন: একটি প্রি-কনফিগারড ইমেজ নির্বাচন করুন যেমন Amazon Linux 2 AMI বা Ubuntu।
    • ইনস্ট্যান্স টাইপ নির্বাচন করুন: আপনার প্রয়োজন অনুযায়ী একটি টাইপ নির্বাচন করুন (যেমন, t2.micro)।
    • ইনস্ট্যান্স কনফিগার করুন: প্রয়োজনীয় সেটিংস পরিবর্তন করুন।
    • স্টোরেজ যোগ করুন: ডিফল্ট স্টোরেজ ব্যবহার করুন অথবা আরও যোগ করুন।
    • ট্যাগ যোগ করুন: আপনার ইনস্ট্যান্সগুলি সংগঠিত করতে ট্যাগ যোগ করুন।
    • সিকিউরিটি গ্রুপ কনফিগার করুন: ট্রাফিক নিয়ন্ত্রণের জন্য নিয়ম সংজ্ঞায়িত করুন।
    • পর্যালোচনা এবং লঞ্চ করুন: আপনার সেটিংস পর্যালোচনা করুন এবং ইনস্ট্যান্স চালু করুন। আপনার প্রাইভেট কী ফাইল সংরক্ষণ করুন।
  4. আপনার ইনস্ট্যান্সে সংযুক্ত হন: SSH বা SSH ক্লায়েন্ট ব্যবহার করে সংযুক্ত হতে নির্দেশনা অনুসরণ করুন।

ধাপ ২: আপনার ইনস্ট্যান্স পরিচালনা করা

  • বন্ধ এবং শুরু: ব্যবহার না করলে খরচ বাঁচাতে ইনস্ট্যান্স বন্ধ করুন।
  • মুছে ফেলা: প্রয়োজন না হলে ইনস্ট্যান্স স্থায়ীভাবে মুছে ফেলুন।

ধাপ ৩: সিকিউরিটি এবং নেটওয়ার্কিং

  • সিকিউরিটি গ্রুপস: অ্যাক্সেস নিয়ন্ত্রণের জন্য নিয়ম সংজ্ঞায়িত করুন।
  • কী পেয়ার্স: ইনস্ট্যান্সে নিরাপদভাবে সংযুক্ত হোন।
  • VPC: আপনার নেটওয়ার্কের উপর আরও নিয়ন্ত্রণের জন্য কাস্টম VPC তৈরি করুন।

অতিরিক্ত বৈশিষ্ট্য

  • ইলাস্টিক লোড ব্যালান্সিং (ELB): আগত ট্রাফিক বিভিন্ন ইনস্ট্যান্সে বিতরণ করে।
  • অটো স্কেলিং: চাহিদার ভিত্তিতে ইনস্ট্যান্সের সংখ্যা স্বয়ংক্রিয়ভাবে সামঞ্জস্য করে।
  • স্পট ইনস্ট্যান্স: কম দামে অপ্রচলিত EC2 ক্ষমতার জন্য বিড করুন।
  • রিজার্ভড ইনস্ট্যান্স: একটি সময়কালের জন্য EC2 ব্যবহার করতে প্রতিশ্রুতির জন্য ডিসকাউন্ট প্রদান করে।
  • সেভিংস প্ল্যান: ব্যবহারের প্রতিশ্রুতি ভিত্তিক সাশ্রয় সহ নমনীয় মূল্য মডেল।
